# Security Policy

## Scope
- **Educational integrity**: All content should uphold factual accuracy and avoid misinformation.
- **Repository protection**: Ensure repository content is free from vulnerabilities, malicious code, or misleading information.
- **Wallet safety**: Provide guidance on how to safely store and manage wallets, including securing private keys and seed phrases.

## Reporting
Report any security issues or vulnerabilities **responsibly and privately** to the project maintainers. Use GitHub issues or contact via email, and do not disclose the issue publicly until resolved.
